# Manifesto: Museum Digital Resource Service (MDRS)

## 1. 核心定义 (Identity)
这是一个**“以文物为中心”的本地化数字圣殿**。
它不是一个通用的文件管理器，而是一个专为博物馆场景打造的、基于“资源即服务（Asset as a Service）”理念的数字资产管理系统。它既是摄影师的工作台，也是研究员的资料库。

## 2. 核心原则 (Core Principles)

### 2.1 绝对本地化 (Local Sovereignty)
* **数据主权：** 所有数据（图片、元数据、模型）必须且只能存储在本地服务器上。
* **零依赖：** 系统必须能在断网环境下完美运行。拒绝任何必需的公有云依赖。
* **文件即真理：** 数据库只是索引，文件系统的层级结构和原始文件才是最终的真理。

### 2.2 资源服务化 (Service-Oriented)
* 系统不应只是“存放”文件，而是要“服务”文件。
* 每一个上传的文物影像，系统都应自动提供服务：自动生成缩略图、自动提取 EXIF、自动关联相关文物记录、自动进行 OCR 或特征识别。
* **输入即处理：** 用户动作应尽量少，后台自动化服务应尽量多。

### 2.3 极简与清爽 (Clean & Crisp)
* **博物馆美学：** UI 风格应如同博物馆展厅一般——安静、留白、聚焦。
* **低噪：** 界面上只保留当前任务最核心的信息。
* **内容优先：** 在任何界面，**文物影像本身**永远是视觉重心，UI 控件应退居次要位置（Dimmed or Hidden）。

## 3. 交互感觉 (The Vibe)
* **响应：** 所有的点击反馈必须是即时的（<100ms），给用户一种“坚固”和“可控”的物理手感。
* **流：** 工作流应该是线性的、无阻碍的。从导入照片到整理归档，中间不应有复杂的弹窗打断。

## 4. 技术底线 (Tech Non-negotiables)
* **性能：** 即使是 100MB 的 TIFF/RAW 文件，预览加载也不能卡顿（利用预生成的代理文件）。
* **标准：** 元数据结构应考虑到未来的扩展性（参考 Dublin Core 或相关博物馆标准），哪怕现在只用简单的 JSON。
